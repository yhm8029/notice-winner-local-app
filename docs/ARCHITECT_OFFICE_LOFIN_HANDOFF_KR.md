# Architect Office / LOFIN Handoff

## 현재 결론

- `architect_office`를 `native_web` 일반 공고문/첨부 fallback으로 채우는 방식은 신뢰하기 어렵다.
- 최근 `43건` 샘플에서 `native_web`가 설계사무소를 제대로 맞힌 사례는 `0건`이었다.
- 반대로 `LOFIN`은 실제 데이터가 있는 케이스가 있었고, 문제는 데이터 부재보다 `검색어(query)` 품질에 가까웠다.
- 결정:
  - `architect_office`는 일반 `native_web` 공고문/첨부 본문 source를 사용하지 않는다.
  - `architect_office`는 계약 소스(`LOFIN`, `EAIS`, `G2B 계약`) 또는 명시적 결과 공고에서만 채운다.

## 현재 문제점

### 1. `architect_office` source 신뢰도 문제

- `native_web` 본문 fallback는 공고문 자격문구를 잘못 잡는 경우가 많다.
- 실제 오인식 예:
  - `참가자격 1) ... 건축사사무소`
  - `도지사에게 건축사사무소`
  - `같은 법 제23조에 따라 건축사사무소`

### 2. LOFIN query 생성 품질 부족

- 같은 케이스라도 아래처럼 query를 잘 줄이면 `LOFIN` hit가 난다.
  - `당진시 육아종합지원센터`
  - `망우본동 생활SOC`
  - `인삼약초특화농공단지 복합문화센터`
- 즉 `project_name_norm` 원문을 그대로 쓰는 것보다, `LOFIN` 검색용 `core query`가 필요하다.

### 3. 일부 miss는 단순 query 문제만은 아님

- 남은 `8건`은 이름을 줄여도 miss였다.
- 이건 아래 가능성이 섞여 있다.
  - 실제 `LOFIN`에 데이터가 아직 없음
  - 날짜 힌트가 어긋남
  - 다른 자치단체/계약명으로 등록됨
  - 관리/운영용역이라 설계사무소 필드 자체가 비적용

## 권장 구현 방향

### 1. `architect_office`에서 `native_web` fallback 제거 또는 강한 제한

- 일반 공고문/첨부 본문으로는 `architect_office`를 채우지 않는 쪽이 안전하다.
- 최소한 `result_notice`가 아닌 일반 공고문 source는 막는 방향이 맞다.

### 2. `LOFIN` 전용 core query 생성

핵심은 화면용 프로젝트명을 바꾸는 게 아니라, 검색용 이름만 별도로 만드는 것이다.

권장 규칙:

- 절차어 제거
  - `설계공모`
  - `제안공모`
  - `간이공모`
  - `국제설계공모`
  - `지명설계공모`
- suffix 정리
  - `건립사업 -> 건립`
  - `조성사업 -> 조성`
  - `구축사업 -> 구축`
- 지역/브랜드/차수/고유명사 유지
- query는 길게 늘리지 말고 `1~2개`만 사용

예:

- `당진시 육아종합지원센터 건립사업 설계공모(제안공모)`
  - `당진시 육아종합지원센터`
  - `육아종합지원센터`
- `망우본동 생활 SOC 건립 설계공모`
  - `망우본동 생활SOC`
  - `망우본동생활SOC`
- `금산인삼약초특화농공단지 복합문화센터 건축설계공모(제안공모)`
  - `인삼약초특화농공단지 복합문화센터`
  - `인삼약초특화농공단지`

### 3. `LOFIN` 조회 파라미터 보강

`WCEGCF` OpenAPI 명세상 아래 파라미터를 같이 넣을 수 있다.

- `laf_cd`
- `ctrt_knd_nm`
- `ctrt_mth_nm`
- `clt_nm`

다음 구현에선 최소한 아래는 같이 넣는 게 좋다.

- `laf_cd`
- `ctrt_knd_nm=용역`

실제 당진/망우본동/금산 사례는 이 조합으로 안정적으로 조회됐다.

## LLM 관련 판단

- 이번 턴에서 `LLM`을 실제 API로 검증한 건 아니다.
- 현재 환경엔 `ANTHROPIC_API_KEY` / `OPENAI_API_KEY`가 없어, 진짜 LLM blind test는 못 했다.
- 다만 이번에 검증된 건:
  - `LLM이 해줬을 법한 짧은 core query`는 실제 `LOFIN`에 먹힌다.
- 그래서 LLM을 쓴다면:
  - `architect_office`를 직접 뽑게 하지 말고
  - `LOFIN` 검색어 후보 `1~3개`를 생성하는 보조 역할로만 쓰는 게 맞다.
