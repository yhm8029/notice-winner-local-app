- 문서 역할: UI/UX 화면 명세서
- 정본 여부: `canonical`
- 기준 커밋: `origin/main` = `eaa3b3e28056aa62182eabe284c8db6ce39b7238`
- 작성일: 2026-04-30
- 상위 기준 문서: [DOCUMENT_GOVERNANCE_MATRIX_KR.md](./DOCUMENT_GOVERNANCE_MATRIX_KR.md)
- 기준 기능 문서: [REBUILD_FUNCTIONAL_SPEC_KR.md](./REBUILD_FUNCTIONAL_SPEC_KR.md)
- 목적: 새 구현팀이 현재 웹 콘솔의 화면 구조, 노출 조건, 버튼, 상태, 오류, 빈 화면을 기존 코드 없이 재현할 수 있도록 고정한다.

## 1. 전체 화면 원칙

1. 단일 웹 콘솔 안에서 로그인 shell, 사용자 모드, 관리자 모드가 전환된다.
2. 기본 진입은 사용자 모드다.
3. 관리자 권한이 있는 사용자에게만 관리자 모드 전환 버튼 또는 관리자 탭을 노출한다.
4. 화면은 업무용 콘솔처럼 조밀하고 스캔하기 쉬워야 한다.
5. 동일 데이터는 목록, 상세, drawer, modal 사이에서 상태가 어긋나면 안 된다.
6. 비어 있는 상태와 오류 상태는 사용자가 다음 행동을 알 수 있게 표시한다.

## 2. 인증 UI

### 2.1 로그인 화면

표시 요소:

1. 서비스명 또는 콘솔명
2. 이메일 입력
3. 비밀번호 입력
4. 로그인 버튼
5. 로그인 실패 메시지 영역
6. 초대 기반 가입 또는 초대 링크 진입 안내

동작:

1. 로그인 버튼 클릭 시 `/api/auth/sign-in` 요청
2. 성공 시 authenticated shell로 전환
3. 실패 시 form 아래에 오류 표시
4. 로딩 중 버튼 disabled
5. 이미 세션이 있으면 home bootstrap 후 사용자 모드로 진입

### 2.2 접근 차단 화면

authenticated 상태이나 유효 membership이 없으면 접근 차단 화면을 보여준다.

표시 요소:

1. 접근 권한 없음 메시지
2. 로그인 계정 이메일
3. 관리자에게 초대를 요청하라는 안내
4. 로그아웃 버튼

## 3. 공통 shell

### 3.1 header

표시 요소:

1. 현재 조직명
2. 사용자명/이메일
3. 역할 badge
4. 사용자 모드/관리자 모드 전환
5. 회원정보 버튼
6. 로그아웃 버튼

관리자 권한이 없으면 관리자 모드 전환은 숨긴다.

### 3.2 전역 상태

화면 전체가 처리해야 하는 상태:

1. 초기 loading
2. bootstrap partial loading
3. authenticated
4. unauthorized
5. network error
6. session expired
7. stale cache with refresh available

## 4. 사용자 모드 홈

사용자 모드 홈은 업무자가 오늘 처리할 프로젝트와 영업 상태를 바로 볼 수 있어야 한다.

필수 영역:

1. 운영 대시보드 요약
2. 실행 생성 패널
3. 최근 실행
4. 최신 리포트/산출물
5. 프로젝트 트래커 목록 또는 보드
6. 내가 진행 중인 영업
7. 회사 전체 진행 중인 영업
8. 전체 영업 대상 프로젝트

### 4.1 운영 대시보드 요약

표시 요소:

1. 최근 실행 상태
2. 성공/실패 요약
3. 트래커 총 프로젝트 수
4. 영업 진행 중 수
5. 다운로드/리포트 job 상태

### 4.2 실행 생성 패널

필수 입력:

1. 실행 유형 선택
2. source/collect mode
3. 날짜/지역/기관/키워드 조건
4. 실행 시작 버튼

버튼 상태:

1. 입력 유효하지 않음: disabled
2. 요청 중: loading
3. 실행 생성 성공: 최근 실행으로 focus 이동
4. 실패: 오류 메시지 표시

### 4.3 최근 실행

각 실행 row/card 표시:

1. run id 또는 짧은 식별자
2. run type
3. status
4. started/finished time
5. progress
6. 최신 로그 일부
7. 상세 보기
8. 취소 버튼
9. 산출물/리포트 버튼

### 4.4 실행 상세

상세 화면 또는 패널 표시:

1. 실행 metadata
2. lifecycle 상태
3. 단계별 진행
4. 로그 스트림
5. SSE 연결 상태
6. 생성된 artifact 목록
7. `tracker_export` child run 상태
8. 오류 JSON 요약

## 5. 트래커 UI

### 5.1 트래커 목록

필수 컬럼:

1. 프로젝트명
2. 기관/발주처
3. 지역
4. 주요 날짜
5. 상태
6. 금액/예상 금액
7. 연락처
8. 관련 공고
9. 영업 상태
10. 수정 상태

필수 상호작용:

1. 검색
2. 정렬
3. 필터
4. 페이지네이션 또는 무한 스크롤
5. 상세 drawer 열기
6. editable field 수정
7. 관련 공고 열기
8. 영업 claim 버튼

### 5.2 트래커 상세 drawer

표시 요소:

1. 프로젝트 기본 정보
2. 원본 값과 수정 값
3. effective value
4. 연락처 resolution summary
5. 관련 공고 목록
6. 영업 상태와 담당자
7. change event history

### 5.3 missing report/cleanup UI

관리자에게만 노출한다.

표시 요소:

1. 누락 필드 유형
2. 대상 프로젝트 수
3. preview 결과
4. conflict 목록
5. apply 버튼
6. apply 결과 요약

apply 버튼은 destructive 성격이 있으므로 확인 대화상자를 거친다.

## 6. 관련 공고 UI

관련 공고는 프로젝트 상세 drawer 또는 별도 panel에서 열린다.

표시 요소:

1. 관련 공고 제목
2. 기관/출처
3. 공고일/마감일
4. 관련도 또는 matching 근거
5. published snapshot 여부
6. 원문 열기 버튼

빈 상태:

1. 관련 공고가 없으면 "관련 공고 없음"을 표시한다.
2. cache가 오래되었으면 새로고침 버튼을 표시한다.

## 7. 영업 UI

### 7.1 내가 진행 중인 영업

각 카드/row 표시:

1. 프로젝트명
2. 담당자 본인 표시
3. 마지막 메모
4. 관련 공고 수
5. 예상/계약 정보
6. 메모 추가 버튼
7. 이관 버튼
8. 해제 버튼
9. 종료 버튼

### 7.2 회사 전체 진행 중인 영업

표시 요소:

1. 담당자
2. 프로젝트명
3. 진행 상태
4. 마지막 메모
5. 시작일/수정일

일반 사용자는 조회 중심이다. 관리자만 강제 이관/강제 해제를 수행할 수 있다.

### 7.3 전체 영업 대상 프로젝트

표시 요소:

1. 미배정 프로젝트
2. 이미 진행 중인 프로젝트
3. 종료된 프로젝트 제외 또는 별도 archive 이동
4. claim 버튼
5. 상세 보기

### 7.4 영업 모달

필수 모달:

1. 메모 입력/수정
2. 이관 대상자 선택
3. 해제 확인
4. 종료 처리
5. 계약 금액 입력

종료 처리 모달은 `close_won`과 `close_lost`를 구분한다.

## 8. 관리자 모드

관리자 모드는 상단 탭 또는 관리자 전용 navigation으로 구성한다.

필수 탭:

1. 조직 운영
2. 사용자/초대
3. 영업 관리
4. 감사 로그
5. Google Sheets
6. 실행/트래커 관리
7. 플랫폼 관리자 도구

현재 구현 기준:

1. 일부 legacy route alias는 호환용으로 유지할 수 있다.
2. 사용자가 직접 접근한 legacy route도 새 관리자 탭으로 redirect 또는 동일 화면 render가 되어야 한다.

### 8.1 사용자/초대 화면

표시 요소:

1. 초대 생성 form
2. 초대 목록
3. 초대 상태
4. 초대 링크/fallback 표시
5. 사용자 목록
6. 역할 변경
7. 소속 상태 변경
8. 계정 상태 표시

### 8.2 감사 로그 화면

표시 요소:

1. 로그인 감사 로그
2. 다운로드 감사 로그
3. 조직 감사 로그
4. sales event 로그
5. 필터와 기간 선택

### 8.3 플랫폼 관리자 도구

표시 요소:

1. 계정 생성
2. 비밀번호 초기화
3. 대상 조직/사용자 표시
4. 실행 결과와 오류 메시지

현재 조직 scope를 벗어나는 전역 조직 전환 UI는 필수 범위가 아니다.

## 9. Google Sheets 관리자 UI

필수 구성:

1. 연결 상태 summary
2. sheet 목록
3. 선택된 sheet table
4. 컬럼 필터
5. 동기화 버튼
6. 마지막 동기화 시간
7. 동기화 job 결과
8. 오류 상세

테이블은 행 수가 많아도 화면이 멈추지 않아야 한다. 컬럼 필터는 사용자가 값 목록을 보고 선택할 수 있어야 한다.

## 10. 다운로드 UI

tracker 다운로드는 job 기반 UI를 제공한다.

필수 상태:

1. 대기
2. 생성 중
3. 완료
4. 실패
5. 만료

완료 시 다운로드 버튼을 표시한다. 실패 시 오류 메시지와 재시도 버튼을 표시한다.

## 11. 오류와 빈 상태

공통 오류 표시:

1. 인증 만료: 재로그인 안내
2. 권한 없음: 역할/조직 확인 안내
3. 네트워크 실패: 재시도 버튼
4. 서버 오류: request id 또는 오류 요약
5. validation 실패: field 단위 메시지

빈 상태:

1. 실행 없음: 실행 생성 유도
2. 트래커 없음: 실행 필요 안내
3. 영업 대상 없음: 필터 해제 안내
4. 관련 공고 없음: 없음 상태 명시
5. Google Sheets 데이터 없음: 동기화 필요 안내

## 12. UI 검수 기준

1. 일반 사용자는 관리자 기능을 볼 수 없다.
2. 관리자는 사용자 모드와 관리자 모드를 오갈 수 있다.
3. 실행 상태는 SSE 또는 fallback으로 갱신된다.
4. tracker 수정 후 목록과 상세 drawer가 같은 값을 표시한다.
5. 영업 claim/transfer/release/close 후 세 목록이 동시에 일관된다.
6. Google Sheets 테이블은 필터/동기화/오류 상태를 표시한다.
7. 모든 destructive action은 확인 단계를 가진다.
